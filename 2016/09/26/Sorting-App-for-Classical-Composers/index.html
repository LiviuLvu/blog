Sorting App for Classical Composers

2016-09-26
#async #list-js #css-libraries #jQery

Getting to know list.js

Finally got it. The connection between list.js and the JSON file is working.
Now it is a lot of fun to just add sorting features through list.js

As I am still learning to work with json and async calls like getJSON() in this case, it took a lot of trial and eror to use both jquery and list.js together, and add the sort and search features.
At this point I am visualising the different ways I could display the Items considering future data to be added to each composer in the list.

But before I get too involved in the layout there are at least two more important features I want to implement. These are “fuzzy search” for easy searching complicated names and a audio sample available for each composer.
It’s a great thing that I dont have to worry about the visual part so much. There are so many great css libraries you can use to make the page look good, and focus mostly on app functionality.

After switching through a few small css libraries

I was wery pleased with the result given by some of the small css libraries. A particular favourite for me is milligram.css
Skeleton.css also works very well out of the box.
For milligram, a particular feature I love is that when using it for this sorting project, combined with list.js, the button that was clicked for sorting, remained in a clicked state, clearly indicating the sorting criteria.

One aspect I am not a fan of in miligram.css is the purple color, but in this case it kind of fits with the classical composers idea.

Finding audio samples for each of the 50 composers took a lot more than I expected, over 2.5 hours. After creating the play link for the audio sample, the user could start multiple sounds at a time. Not good! This was another tricky step for me and I looked at a lot of examples before figuring out a solution.

After all functionality was in place, it was time to make this look good, both on mobile and desktop. This is one of the parts I enjoy most, as the project is close to completion.

Most difficult decision

for the layout was to display the data in a modal, when the user clicks a link to see more info about a profile. On desktop it is easy, but on mobile it has to be easy to close.

Getting the correct data to be displayed in the modal was another difficult part but learned a lot in the process.
In the end the app looks quite nice and I like the way it behaves. I am sure there is a lot that could be improved so let me know your thoughts on this.

Feel free to fork the project on git and improve it or make your own version, and please let me know what you built.
