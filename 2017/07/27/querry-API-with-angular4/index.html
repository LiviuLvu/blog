Query API with Angular 4

2017-07-27
#angular4 #typescript #angular animations

Rambling about building an app with angular to show some data from Marvel API.

Received a challenge to build a small app that connects to the Marvel API. The limits imposed are to build it using angular (any version I choose), provide unit tests, provide a search function.
Needless to say I failed to build the app.

I had no idea how to use angular but did not expect to take me more than a week to learn. Basics are simple to learn in 2 days, but then I hit a brick mind wall when reaching the area of http calls, routing and testing.

Two years back after doing a lot of tutorials, I still could not build an advanced app with angular js (ver 1). That time I was also very novice in javascript. Made the mistake of investing time in frameworks, instead of learning true javascript basics. Now I thought Angular should be easy to use, given my understanding of javascript, but it was not so. After two weeks and a lot of tutorials and reading, I still struggle.

Part of the added difficulty is that I had no understanding about writing type script. Tough type script seemed difficult to get at first, I am starting to like it more, the more I learn about it and use it. Combined with visual studio auto complete and suggestions it saves a ton of spelling errors and faulty logic (my logic that is).

Completed a few tutorials on basics, hoping it will get me closer to understanding the parts that make an angular app, and it did help. The only thing missing now is to understand how to modify the params for the http call in order to get the data from the API that I need. In this case Marvel comics.

What is unusual for me about this api call is the combination of hashed passwords and time stamp.
The api takes three params: the link + time stamp + hash(time stamp + private + public password). That means I need 2 functions. One to create the time stamp, another to create the hash combination. At least that is how I see one way of providing the params for the http call. Never mind, for now I will leave those hard coded.

I tweaked a bit on the css and spent a lot of time with the angular animations module to understand it and build the animations I wanted to have in the UI. The transitions help a lot with the user experience.

Now it is done, but I need to get going with testing.
