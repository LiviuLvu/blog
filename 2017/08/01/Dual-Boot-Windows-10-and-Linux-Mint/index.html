Dual Boot Windows 10 and Linux Mint

2017-08-01
#linux-mint #linux #dual-boot-linux-windows

This weekend I installed Linux Mint along side with Windows 10, which was already installed. This was my long desire for a few weeks, but I always delayed installation due to the complicated setup and multiple different settings both in UEFI and formatting.

There are a lot of old tutorials for windows 7 that do not work for win 8 or 10. Somehow I suspect I’m not the only one that had a hard time dual booting windows10 with a linux distro, so I decided this post will be a short one outlining the most difficult parts of the setup.
The goal is to add a linux distro when you already have windows 10 installed.

Preparation setup is easy

After building a bootable USB with Rufus, you need to create space for Linux on the drive.
Shrink the windows partition with disk management tool and leave some 100GB of unused space (probably less is needed, but that is how i went). There is also the option to make this partition with the linux manager, but this seemed more convenient and safe.

Incredible USB booting difficulties

Most of the day was spent reformatting the USB with all settigns and multiple distros, and none would boot. After destroying one USB and buying another, and lots of bios / UEFI tinkering, I discovered that activating the DD compatibility option in Rufus made the USB bootable.

Entering the BIOS/UEFI is another discovery adventure. My laptop uses F2 to enter BIOS/UEFI, and F12 to display startup options, but there are lots of others for different computers.

Installing Linux Mint

Great joy after making the usb bootable. Now I was on familiar ground, I’ve done this before and watched a lot of tutorials about partitions schemes.
After trying Ubuntu, Solus and Mint, in the end I liked the Mint distro the most. My partition structure is as follows:

  1. BOOT — 500MB — EXT4
  2. SWAP — (minim your RAM size) — EXT4
  3. ROOT — 60GB — EXT4

There are a lot of tutorials where the BOOT partition is not created, but this works for me. SWAP is recommended to be 2x the RAM capacity.

Boot manager problems

Very important setting is the “Device for boot loader”. Since I made a separate BOOT partition, I selected that.

This last setting is by far the most confusing part since windows 10 does not use BIOS, but UEFI. This has protection mechanisms that make dual boot a lot harder to setup.
Some people seem to leave the setting untouched, while others select the “windows boot manager” partition. Make sure you have a windows recovery USB ready in case you get it wrong here and need to repair the boot manager.

Finally after the install was done, booted computer but this fella enters directly in Windows as if nothing happened.
Luckyly I remembered a setting in UEFI that allows to select trusted files to run when starting the machine. It is called: Select an UEFI file as trusted for executing
See this video for how to find it.
In there you navigate to HDD0/EFI/ubuntu/ or something similar. Now you should see a list of files, of wich one should be shim64.efi. Add this file, save, exit and enter bios again.

Read more about shim64.efi:
“… is a relatively simple program that provides a way to boot on a computer with Secure Boot active”

You will see the new addition listed in the boot order menu.
You need to move it to top priority.

Other options to access the ubuntu boot loader are:

  • use F12 boot menu to show boot options
  • from Windows10, click restart while holding shift

The Mint distro feels very smooth. I installed node.js, visual studio, GIT and some more small programs. I just love it!

Ok, hope this helps you with your install. Best of luck!
